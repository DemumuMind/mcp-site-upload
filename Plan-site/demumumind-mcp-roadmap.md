### Глава 1: Технический стек (Выбор инструментов)

Чтобы сделать интерфейс таким же чистым и современным (как на скриншотах) и обеспечить быструю работу, рекомендую следующий "Modern Stack":

*   **Frontend:** **Next.js 15 (App Router)** — для SEO (очень важно для каталога) и скорости.
*   **Styling:** **Tailwind CSS** — для верстки.
*   **UI Kit:** **Shadcn/ui** — это *ключевой момент*. Скриншоты очень похожи на компоненты Shadcn (карточки, бейджи, кнопки, темная тема). Это сэкономит 80% времени на дизайн.
*   **Backend & Database:** **Supabase** (PostgreSQL). На скриншоте прямо написано "powered by Supabase". Это даст базу данных, аутентификацию и Realtime-функции "из коробки".
*   **Deployment:** **Vercel**.
*   **Validation:** **Zod** (для валидации форм регистрации серверов).

### Глава 2: Проектирование Базы Данных (Supabase)

Нам нужна реляционная структура. Основные таблицы:

1.  **`servers`** (Главная таблица)
    *   `id` (uuid)
    *   `name` (text) — например, "Amplitude", "Apify"
    *   `description` (text) — краткое описание
    *   `url` (text) — адрес сервера (endpoint)
    *   `category` (text/enum) — Analytics, Project Management и т.д.
    *   `auth_type` (enum) — OAuth, API Key, Open (No Auth)
    *   `maintainer_name` (text)
    *   `maintainer_email` (text, private)
    *   `is_verified` (boolean) — галочка верификации
    *   `status` (enum) — active, pending, dead
    *   `created_at` (timestamp)

2.  **`tools`** (Инструменты внутри сервера - опционально для MVP)
    *   Если вы хотите показывать, какие именно функции (`get_analytics`, `add_task`) есть у сервера, нужна отдельная таблица, связанная с `servers`.

### Глава 3: Разработка Frontend (Постранично)

#### 1. Главная страница (Catalog / Home)
*   **Header:** Логотип, поиск (Search bar), кнопка "Submit Server".
*   **Hero Section:** Заголовок "DemumuMind MCP", подзаголовок, статистика (150+ Servers, 500+ Tools — как на скрине 3).
*   **Search & Filter:**
    *   Инпут для текстового поиска.
    *   Фильтры (Dropdown) по категориям (Finance, Weather, Dev).
    *   Фильтры по типу доступа (Open, Paid, OAuth).
*   **Server Grid (Сетка карточек):**
    *   Использовать CSS Grid.
    *   **Карточка сервера (Card Component):**
        *   Иконка/Лого.
        *   Название + Бейдж (Lock icon для OAuth / Unlock для Open).
        *   Тег категории (синий бейдж).
        *   Поле с URL (с кнопкой "Copy to clipboard" — иконка копирования).
        *   Кнопки действий: "Visit" (ссылка на доку) и "Tools" (попап со списком инструментов).

#### 2. Страница регистрации (Register Your Server)
*   Это форма (как на скрине 2).
*   Поля: Name, URL, Category, Authentication, Maintainer info, Description.
*   Логика: После отправки запись падает в базу со статусом `pending`. Админ (вы) должен её одобрить.

#### 3. Страница деталей (опционально)
*   `/server/[slug]` — отдельная страница для SEO, где расписаны все методы API этого сервера, примеры конфигов для Claude Desktop или других LLM-клиентов.

### Глава 4: Реализация функционала (Logic)

#### 1. Поиск в реальном времени
Используем возможности Supabase (Postgres Full Text Search) или простую фильтрацию на клиенте (если серверов < 500).
*   *Логика:* Пользователь вводит "Asana" -> список фильтруется мгновенно.

#### 2. Проверка доступности (Health Check) — Killer Feature
Чтобы ваш каталог был *лучше* других, напишите скрипт (Cron job via GitHub Actions или Vercel Cron), который раз в сутки пингует URL серверов.
*   Если сервер отвечает 200 OK — ставим зеленый индикатор.
*   Если 404/500 — красный.

#### 3. Автоматический парсинг (Advanced)
MCP — это протокол. Вы можете написать скрипт, который стучится на указанный URL и запрашивает список инструментов (`list_tools`).
*   *Результат:* Пользователь вводит URL при регистрации -> Сайт сам подтягивает описание и список доступных функций. Это будет "Wow-эффект".

### Глава 5: Дизайн и UX (Как на скринах)

*   **Цветовая палитра:**
    *   Background: Очень темный синий (почти черный) `#0b0f19` или Slate-950.
    *   Card Background: Чуть светлее, с легким бордером `#1e293b`.
    *   Accent Color: Ярко-голубой (Electric Blue) для кнопок и активных элементов.
*   **Шрифты:** Inter или Geist Sans (стандарт для Vercel/Modern web).
*   **Микро-взаимодействия:**
    *   Hover-эффекты на карточках (легкое свечение).
    *   Тост-уведомление (Toast) при копировании ссылки ("Copied to clipboard!").

### Глава 6: План запуска и SEO (Marketing)

1.  **Наполнение контентом (Cold Start):**
    *   Не запускайте пустой сайт. Спарсите существующие репозитории (например, `modelcontextprotocol/servers` на GitHub).
    *   Добавьте первые 50 серверов вручную.

2.  **SEO оптимизация:**
    *   Динамическая генерация `sitemap.xml`.
    *   Правильные мета-теги для каждого сервера: "Asana MCP Server for Claude Desktop".

3.  **Дистрибуция:**
    *   Опубликовать на Hacker News, Reddit (r/LocalLLaMA, r/ClaudeAI, r/OpenAI).
    *   Сделать Pull Request в официальную документацию MCP (если возможно), чтобы они на вас сослались.
